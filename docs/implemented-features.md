# Реализованные функции

## Аутентификация

Система поддерживает регистрацию и вход по JWT-токену.

После регистрации пользователю автоматически назначается роль `Applicant`.

При входе пользователь получает JWT-токен сроком действия 7 дней. В токен записываются идентификатор пользователя, логин, ФИО и роль.

Доступные роли:

* Applicant
* Manager
* Admin
* Director

Для защищённых маршрутов используется авторизация по ролям. При каждом запросе дополнительно проверяется существование пользователя. Если пользователь был удалён, система возвращает ошибку авторизации.

## Работа с заявками

Пользователь может создать заявку, указав название, описание, направление и формат обучения. Автор заявки определяется автоматически по данным токена. Новая заявка получает статус `New`.

Требования к данным:

* Название: от 3 до 200 символов
* Описание: от 10 до 2000 символов

Доступен просмотр списка заявок с фильтрацией по:

* статусу
* направлению
* формату
* дате создания

Обычные пользователи видят только свои заявки. Руководители и администраторы могут просматривать все заявки.

Менеджер может назначить ответственного за заявку. Если заявка находится в статусе `New`, она автоматически переводится в статус `InProgress`.

Менеджер также может изменять статус заявки. Для перевода в статусы `Approved`, `Rejected` или `Completed` должен быть назначен ответственный сотрудник.

## Статусы заявок

Поддерживаются следующие статусы:

* New
* InProgress
* NeedsInfo
* Approved
* Rejected
* Completed

Все изменения статусов сохраняются в истории. Для каждого изменения фиксируются старый статус, новый статус, пользователь и время изменения.

## Комментарии

Пользователи могут оставлять комментарии к заявкам.

Пользователь с ролью `Applicant` может работать только со своими заявками. Пользователи с ролями `Manager`, `Admin` и `Director` могут комментировать любые заявки.

Длина комментария должна быть от 1 до 1000 символов.

## Справочники

Система содержит справочники направлений обучения и форматов обучения.

Все пользователи могут просматривать данные справочников.

Создание, изменение и удаление доступны только администраторам.

Также доступен справочник статусов заявок в режиме только для чтения.

## Аналитика

Раздел аналитики доступен ролям `Director` и `Admin`.

Отображаются:

* общее количество заявок
* общее количество пользователей
* распределение заявок по статусам
* распределение по направлениям
* распределение по форматам обучения

На странице статистики доступны:

* ключевые показатели
* график динамики заявок
* популярные направления
* статистика по форматам и статусам

## Пользователи

Менеджеры, администраторы и директора могут получать список пользователей.

Поддерживается фильтрация по роли.

Для каждого пользователя отображаются:

* идентификатор
* ФИО
* роль

## Аудит действий

Система ведёт журнал изменений.

Для каждого действия сохраняются:

* пользователь
* тип действия
* объект изменения
* идентификатор объекта
* дата и время

Фиксируются операции создания, изменения и удаления данных.

## Логирование

Все основные действия и ошибки записываются в журналы.

Логи сохраняются:

* в консоль
* в файлы

Файлы логов создаются ежедневно и хранятся 14 дней.

## Обработка ошибок

Система обрабатывает основные типы ошибок и возвращает ответ в формате JSON.

Поддерживаются следующие коды:

| Ошибка                    | HTTP код |
| ------------------------- | -------- |
| Некорректный запрос       | 400      |
| Нет доступа               | 401      |
| Внутренняя ошибка сервера | 500      |

Все ответы содержат поле `message` с описанием ошибки.

## База данных

В качестве базы данных используется PostgreSQL 15.

Особенности:

* работа через Entity Framework Core
* автоматическое применение миграций при запуске
* идентификаторы формата UUID
* уникальные логины и email пользователей

Основные таблицы:

* Users
* Applications
* Directions
* TrainingFormats
* Comments
* StatusHistories
* AuditLogs

## Пользовательский интерфейс

Клиентская часть построена на React, TypeScript и Tailwind CSS.

Реализованы страницы:

* вход в систему
* список заявок
* создание заявки
* просмотр заявки
* справочники
* пользователи
* статистика

В интерфейсе используются:

* хранение данных авторизации в localStorage
* автоматическая подстановка JWT-токена в запросы
* адаптивная верстка
* навигация по ролям
* защищённые маршруты

После входа пользователь автоматически перенаправляется на доступный ему раздел системы.

## Инфраструктура

Проект запускается через Docker Compose.

Используются три сервиса:

1. PostgreSQL
2. Backend на .NET 8
3. Frontend на Nginx

Для данных и логов используются отдельные тома.

Настройки окружения хранятся в файле `.env`.

## CI/CD

Для автоматизации используется GitHub Actions.

При каждом запуске выполняются:

1. Сборка backend-приложения
2. Запуск тестов
3. Сборка frontend-приложения
4. Проверка кода линтером

## Тестирование

Покрыты основные сценарии работы:

* авторизация
* работа с заявками
* справочники
* пользователи
* аналитика
