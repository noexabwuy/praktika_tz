# Известные ограничения

## Что не реализовано

В текущей версии отсутствуют следующие возможности:

* экспорт заявок в CSV и Excel
* уведомления о смене статуса по email, в Telegram или внутри системы
* обновление списка заявок в реальном времени
* отдельные мобильные и настольные приложения

Эти функции не входили в рамки минимальной рабочей версии проекта.

## Технические ограничения

На данный момент в проекте отсутствуют фронтенд-тесты. React-компоненты, страницы и пользовательские хуки не покрыты автоматическими тестами. Тестирование реализовано только для серверной части.

Также отсутствуют интеграционные тесты. Все существующие тесты работают через InMemory Database и не проверяют полный цикл взаимодействия приложения.

Страница статистики строит график динамики заявок на основе всех данных, полученных с сервера. При значительном объёме данных может потребоваться перенос расчётов на серверную сторону.

Для локального запуска используется значение JWT-секрета по умолчанию, если оно не переопределено через переменные окружения. В рабочей среде необходимо использовать собственный секретный ключ.

Метод получения списка заявок не поддерживает пагинацию. Сейчас API возвращает весь набор данных одним запросом. При увеличении количества записей потребуется постраничная загрузка.

## Особенности тестовых данных

Тестовый пользователь `kuznetsov` с ролью менеджера не назначен ни на одну заявку. Его можно использовать для проверки механизма назначения ответственного.

Пользователи `fedorova`, `popov` и `novikova` не имеют созданных заявок. Их можно использовать для проверки создания новых заявок.

## Безопасность

Для упрощения разработки и демонстрации CORS настроен без ограничений по источникам запросов.

Перед использованием системы в рабочей среде необходимо ограничить список разрешённых доменов.
